<objective>
Add `SectionNode.collapsible?: boolean` — a pure client-side aesthetic disclosure primitive using native `<details>` / `<summary>`. Open/closed state is DOM-local; it survives server-driven re-renders via a new in-renderer snapshot/restore pair (same pattern the renderer already uses for `focusId` + `scrollMap`). The wire stays additive: omitted = today's `<section>` behavior, byte-identical. Ships as a lockstep npm + NuGet 1.1.0 → 1.2.0 minor bump.

Purpose: gives apps a built-in way to defer secondary content (e.g. HelpDesk "Agent Notes" beneath the ticket header) without inventing per-app JS, custom CSS, or a server-driven open/close protocol. The conceptual model — "the server has no business knowing what's expanded" — mirrors the framework's existing precedent that draft text values in form inputs also don't round-trip.

Output: SectionNode gains `collapsible?: boolean` (+ optional `id?: string` for stable preservation keying); the browser adapter renders `<details>/<summary>` when set; default.css styles `.vms-section__summary` to match `.vms-section__heading` typography with the framework's `:focus-visible` idiom; HelpDesk's "Agent Notes" demos the feature on both backends; lockstep 1.2.0 release lands on the current branch (no push, no auto-branch — per the working-agreement override).
</objective>

private section(n: SectionNode, parent: HTMLElement, on: (a: ActionEvent) => void): void {
  const el = document.createElement("section");

  this.kids(n.children, el, on);
  parent.appendChild(el);
}

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
