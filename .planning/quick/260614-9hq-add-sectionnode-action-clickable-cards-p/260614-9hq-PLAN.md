<objective>
Add `SectionNode.action` — a click-anywhere clickable-card primitive that mirrors `TableRow.action` (restored in 1.1.0) at the section level. Lockstep TS + .NET wire field; tree-validation rejects two invalid combos at build time with `invalid_tree`. Closes the unfixed half of issue #19 — see issue #20 for the full design contract.

Purpose: A `SectionNode variant: "card"` is styling-only today; making a card clickable requires an inner `ButtonNode` that splits the affordance from the surface. This adds the missing primitive without inventing new node types — same idiom, one level up the tree.
