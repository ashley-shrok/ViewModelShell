<objective>
Fix issue #17: SectionNode with layout="cards" or layout="split" silently collapses to a 1-column flex stack because the base `.vms-section { display: flex }` rule is declared LATER than the `.vms-section--cards` / `--split` modifier rules in default.css. Under equal specificity, later wins — so the base flex always shadows the grid modifiers. PageNode is unaffected because `.vms-page` is declared in the correct order.

Purpose: An explicit, server-declared layout intent (`layout: "cards"` / `"split"`) currently does nothing on SectionNode. This is silent — no error, no warning — and breaks the dashboard-tile / list-detail pattern documented in AGENTS.md "Design system" as a canonical worked example. Existing class-emission tests passed because the bug is in the computed CSS cascade, not the emitted className. We fix the cascade AND add a regression test that actually loads the real stylesheet into jsdom and asserts `getComputedStyle().display`, closing the test-coverage gap that let this slip through.

Also: add `min-width: 0` on direct children of grid containers so wide media (full-width images, long unbroken strings) shrinks to the track instead of blowing the grid out and collapsing it back to one column even after the cascade fix.

Output:
- viewmodel-shell/styles/default.css: base `.vms-section` block moved above its modifiers, mirroring `.vms-page` structure; new `min-width: 0` rule for grid-child overflow protection with explanatory comment.
- viewmodel-shell/test/theme-modifiers.test.ts: new describe block loading the real default.css into jsdom and asserting computed `display` values.
- viewmodel-shell/package.json: version 1.0.0 → 1.0.1.
- CHANGELOG.md: new "1.0.1 — 2026-06-08" entry at top.
- Three atomic commits on the current branch (main).
</objective>
