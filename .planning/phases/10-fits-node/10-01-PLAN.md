<objective>
Add the `fits` node — the SwiftUI `ViewThatFits` port — to the wire on BOTH backends, plus the measure-and-pick renderer in `browser.ts`. The renderer measures the available container width via a `ResizeObserver` and synchronously picks the FIRST child whose intrinsic size fits the axis (no overflow, 1px tolerance), falling back to the LAST (guaranteed-fits) child. In jsdom/SSR/detached contexts (`clientWidth === 0`) it renders only the LAST child. ResizeObservers are tracked per-render and disconnected at the top of `render()` so they never leak. This is the wire + CSS + renderer + lifecycle + structure/fallback-test layer.

Purpose: Deliver FITS-01 (the responsive-selection node + measure-and-pick renderer) and FITS-03 (the byte-identical TS/.NET wire type with its `[JsonDerivedType]` discriminator). The `fits` node is the ONE primitive that is not pure CSS — it requires real layout measurement, which lives entirely in `BrowserAdapter` (`browser.ts`), never in platform-agnostic core (`index.ts`).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| server→client wire | The `fits` node's `axis` value and `children` cross the wire; the renderer measures and renders candidate children client-side. No `axis` value is interpolated into a class name (the `.vms-fits` class is static), so there is no class-injection surface from `axis`. |
| client layout measurement | `fits()` runs DOM measurement + a ResizeObserver in the browser only; no untrusted input reaches measurement beyond the (server-authored) view tree, which the server already fully controls. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Tampering | browser.ts fits() rendering of candidate children | accept | Candidates are ordinary ViewNodes rendered via the existing `this.node(...)` path — same trust posture as any other container's children. A hostile server already fully controls the view tree (it IS the view); `fits` adds no new injection capability. `axis` is a closed union (compile error off-union on honest backends) and is NOT interpolated into any class name, so there is no class-string-injection surface (unlike the `vms-arrange--{value}` precedents, the `.vms-fits` class is static). No runtime sanitization added. |
| T-10-02 | Denial of Service | browser.ts ResizeObserver lifecycle | mitigate | Observers are tracked in `fitsObservers` and ALL disconnected + cleared at the top of every `render()` before the tree is rebuilt, so a long-lived session cannot accumulate orphaned observers across re-renders (leak prevention). The container being block/full-width makes its observed width parent-driven, preventing a measure→resize→measure feedback loop that would otherwise spin the observer. Verified by the jsdom disconnect-on-re-render test (Task 3). |
| T-10-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs in this plan — pure source edits to existing files; no new dependencies. |
