- **The label is view, not state:** `bind` holds the id; the node carries `selected?: [{value,label?,type?}]`, server→client only, recomputed every render, never trusted inbound — so a form that loads with a reference already set renders its label with no search having occurred, and filtering the candidate list can never erase the selection.
- **The live-query lane** — VMS's first live-query dispatch cadence: `searchAction` dispatches debounced (~300ms) on keystroke, renderer-FORCED onto the v4.2 non-blocking lane (`blocking` is meaningless on `searchAction`), with `searchBind` round-tripping the query; stale/out-of-order responses never clobber a newer one. No min-chars gate.
- **`allowCustom`** — the explicit declared custom-entry axis; `allowCustom:true` with no candidates is a free-form tags input (this supersedes the parked `inputType:"tags"` design). Invented values stay homogeneous objects — never bare strings.
- The a11y baseline for chips (item-specific remove labels, roving tabindex, add/remove announced with the running count, focus-after-removal never landing on `<body>`) and the persistent live region.
- **`agent-skill.md` now documents the picker as a first-class public protocol.**
- Additive; existing apps and wire-driving agents byte-unchanged; **the wire token stays `viewmodel-shell/1.0`**.
