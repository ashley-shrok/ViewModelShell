<objective>
Add the framework-owned categorical chart palette — 8 tokens `--vms-chart-1 … --vms-chart-8` — to
`styles/default.css` `:root` AND every theme file under `styles/themes/`, and hand-verify each slot's
contrast against the chart plot background. This is the theme-token color layer that the browser
adapter (18-03) cycles per series (per slice for pie/donut). It is a pure-CSS plan with NO overlap on
the wire-type or renderer files, so it runs in parallel with 18-01 in wave 1.

Purpose: color for charts stays framework-owned and theme-token-driven — ZERO raw color on the wire —
exactly like buttons/badges. A curated 8-slot palette gives readers distinguishable series without the
app hand-picking hues; a consumer who wants brand hues retunes these tokens via the existing `--vms-*`
override seam.

Output: `--vms-chart-1..8` in 13 CSS files (default + 12 themes), each slot distinguishable and legible
on the plot background, plus a recorded contrast/distinguishability check in the summary.
</objective>

<notes>
- `styles/default.css` :root holds the base token set (accent/error/warning/success/info + surface/bg/text).
  The theme files override a subset; the adapter reads whatever wins after the cascade. So the 8 chart
  tokens must exist in default.css (the always-present baseline) AND be RE-tuned per theme (each theme's
  hue family + light/dark surface differs, so a slot legible on the light-amber surface may not be on the
  dark-teal surface — that is precisely why every theme gets its own values).
- The chart CANVAS is transparent; the plot background is the container behind it. In the demos/verification
  a chart sits inside a `variant:"card"` section (`--vms-surface`) or a plain page section (`--vms-bg`), so
  BOTH `--vms-surface` and `--vms-bg` are the plot backgrounds to check each slot against.
- LOCKED (do NOT do): do NOT wire chart-palette pairs into `scripts/check-aa-contrast.mjs` — that gate
  covers a FIXED pair-set by design (banked lesson). Contrast here is a HAND-CHECK acceptance criterion,
  recorded in the summary, using a throwaway measurement script — not a permanent gate addition.
- Palette slots are NON-TEXT accents (fills/strokes), so the legibility bar is distinguishability from the
  background and from each other — use the NON-TEXT threshold (~3:1 vs background) as the floor, NOT the
  4.5:1 text bar.
</notes>
</context>
