<objective>
Reshape the published `ChartNode` wire type — in BOTH backends — from the 4.1 single-series
`{ kind?:"bar"; points: ChartPoint[]; title?; tone? }` to the multi-series-native base-set shape
`{ kind?:"bar"|"line"|"area"|"pie"|"donut"; labels: string[]; series: ChartSeries[]; stacked?; title? }`
with `ChartSeries = { name; data: number[]; tone? }`. Retire `ChartPoint` for category charts. This
plan owns ONLY the wire contract + its serialization rules + a .NET omission unit test. It is the
interface every downstream plan (renderer, TUI, parity) builds against, so it lands first.

Purpose: this is the load-bearing breaking reshape that the whole `5.0.0` milestone rests on; taken
now because zero consumers have shipped a chart (the free-reshape window). Getting the optional-field
wire rules exactly right here is what keeps TS/.NET byte-aligned and strict consumers happy.

Output: reshaped `viewmodel-shell/src/index.ts` + `viewmodel-shell-dotnet/ViewModels.cs`, a new
`.NET` serialization test, and a note in the summary that this DELIBERATELY breaks the FeatureProbe
demos + Showcase + chart tests (repaired by plans 18-03 / 18-05) — expected transient red between plans.
</objective>

```typescript
export interface ChartSeries {
  /** Series name — rendered in the legend and read by agents to identify the series. */
  name: string;
  /** Values aligned by index to the chart's `labels`: data[i] is the value at labels[i]. */
  data: number[];
  /** OPTIONAL semantic tone from the existing closed tone axis. When set, this series is drawn in the
   *  theme's tone token (danger→--vms-error, etc.) instead of the next categorical-palette slot.
   *  For MEANING, not decoration. Omitted → framework assigns the next --vms-chart-N slot. NO raw color. */
  tone?: "danger" | "warning" | "success" | "info";
}

export interface ChartNode {
  type: "chart";
  /** CLOSED union; OMITTED = "bar". Widened additively later (scatter, …). */
  kind?: "bar" | "line" | "area" | "pie" | "donut";
  /** Shared category axis. labels[i] is the category for every series' data[i]. */
  labels: string[];
  /** One or more series over the shared labels. Single-series = exactly one entry. */
  series: ChartSeries[];
  /** bar/area only: stack series instead of grouping side-by-side. Omitted/false = grouped. */
  stacked?: boolean;
  /** Optional chart title rendered above the plot. */
  title?: string;
}
```
`ChartPoint` interface is DELETED. `ChartNode` stays in the `ViewNode` union (index.ts ~L154, unchanged).

.NET — `viewmodel-shell-dotnet/ViewModels.cs` (replaces the current ChartPoint + ChartNode records at ~L797–804):
```csharp
public record ChartSeries(
    string Name,
    IReadOnlyList<double> Data,
    [property: JsonIgnore(Condition = JsonIgnoreCondition.WhenWritingNull)] string? Tone = null
);

public record ChartNode(
    IReadOnlyList<string> Labels,
    IReadOnlyList<ChartSeries> Series,
    [property: JsonIgnore(Condition = JsonIgnoreCondition.WhenWritingNull)] string? Kind = null,
    [property: JsonIgnore(Condition = JsonIgnoreCondition.WhenWritingDefault)] bool Stacked = false,
    [property: JsonIgnore(Condition = JsonIgnoreCondition.WhenWritingNull)] string? Title = null
) : ViewNode;
```
The `[JsonDerivedType(typeof(ChartNode), "chart")]` discriminator at ViewModels.cs ~L397 stays unchanged.

RULE SOURCES (do not re-derive — apply):
- AGENTS.md gotcha #8 / the ViewModels.cs file-header rule: nullable wire members carry
  `WhenWritingNull`; an optional non-nullable bool whose `false` means "absent/unset" carries
  `WhenWritingDefault`. `stacked` is exactly that inverted-usual bool (TS `stacked?` omit=false) → so
  `bool Stacked = false` + `WhenWritingDefault`. `Labels`/`Series`/`Name`/`Data` are REQUIRED (no ignore).
- `Data` is `double` (mirrors TS `number`) — whole-number fixtures keep the wire integer-identical.
