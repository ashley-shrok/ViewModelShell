and the expected `parity/fixtures/feature-probe.json` — from the retired single-series `points` case to

not the other fails the build. And per the AGENTS.md maintainer rule, a change to the wire shape MUST
update `agent-skill.md` (then re-copy to `AgentSkill.md`) in the same change, so wire-driving agents
aren't handed a doc describing a retired `points` shape.

and `bun run parity/run.ts` green (which includes `check-skill.ts`).
