<objective>
Reshape the TUI adapter's `ChartView` (`viewmodel-shell/src/tui.tsx`) from the retired single-series
`points[]` degradation to the multi-series `labels[]` + `series[]` shape, and add a TUI test proving
the reshaped ChartNode renders legibly without throwing. The TUI is `@experimental`; the only
requirement (CHARTBASE-05) is that the reshaped node degrades legibly (printed series / ASCII) and
never crashes the non-browser target.

Purpose: keep the non-browser front-end target honest — a terminal has no canvas, but a ChartNode is
structured data, so it prints as a legible label/series/value/ASCII-bar rendering. This plan depends
only on the 18-01 wire type (not the browser renderer or palette).

Output: the reshaped `ChartView` + a `test/tui-chart.test.ts`, with the TUI suites green under vitest.
</objective>
