<objective>
Widen the browser adapter's lazy/optional Chart.js binding (`chart()` + `loadChart()` in
`viewmodel-shell/src/browser.ts`) from the single-series bar to the full base set: multi-series
bar (grouped + `stacked`), `line`, `area` (=line+fill), `pie`, and `donut` (=pie+center cutout),
colored by the `--vms-chart-1..8` palette (per series for bar/line/area, per slice for pie/donut)
with an optional per-series `tone` override, and an auto legend from `series[].name`. Update the
jsdom chart tests + the missing-dep fail-loud test to the new shape, and repair the Showcase demo
chart so the package + demo compile.

Purpose: this is the pixels layer of the reshape — everything the reader/operator sees. It consumes
the 18-01 wire type and the 18-02 palette tokens. Core (`index.ts`), .NET, and bun backends stay
dependency-free: chart.js remains a private, dynamically-imported adapter detail.

Output: the widened `chart()`/`loadChart()`, updated `chart.test.ts` + `chart-missing-dep.test.ts`,
and a reshaped Showcase chart example — all compiling, with `npx vitest run` green for the chart suites.
</objective>

ChartNode (viewmodel-shell/src/index.ts, from 18-01):
  { type:"chart"; kind?:"bar"|"line"|"area"|"pie"|"donut"; labels:string[]; series:{name:string;data:number[];tone?:"danger"|"warning"|"success"|"info"}[]; stacked?:boolean; title?:string }

Palette tokens (from 18-02, present in default.css + every theme): --vms-chart-1 … --vms-chart-8.
Tone tokens (existing): danger→--vms-error, warning→--vms-warning, success→--vms-success, info→--vms-info.

Current renderer to widen — browser.ts:
  - chart(n, parent): builds a Chart.js `config` and manages the persistent `chartInstances` map
    (keyed by title-ordinal), reuses the canvas across renders, calls .update() to redraw in place,
    and render() mark-sweeps/destroys dropped instances. The tone→token color read via getComputedStyle
    is already the pattern (browser.ts ~L568–590). Grid/tick colors already track --vms-border/--vms-text-muted.
  - loadChart(key, config): dynamic `import("chart.js")` then `Chart.register(BarController, BarElement,
    CategoryScale, LinearScale, Tooltip)`. WIDEN the registration.
</interfaces>

<rendering_spec>
LOCKED per-kind behavior (design of record):
- kind omitted or "bar": Chart.js type "bar". Multiple series → multiple datasets (grouped). If `stacked`,
  set `scales.x.stacked = scales.y.stacked = true`.
- kind "line": Chart.js type "line", one dataset per series, `fill:false`.
- kind "area": Chart.js type "line" with `fill:true` (line + fill) per dataset. `stacked` applies (stacked area).
- kind "pie": Chart.js type "pie", SINGLE series = series[0]; `data.labels = labels`, one dataset whose
  `data = series[0].data` and `backgroundColor` is an array cycling the palette PER SLICE (per label).
  If `series.length > 1`, render series[0] and emit ONE dev-console warning (do NOT throw). `stacked` ignored.
- kind "donut": same as pie with `type:"doughnut"` (center cutout). Chart.js's doughnut IS the donut.
- Palette assignment for bar/line/area: dataset i → `--vms-chart-((i mod 8)+1)`, UNLESS that series has a
  `tone`, in which case use the tone token. For pie/donut: slice j → `--vms-chart-((j mod 8)+1)`.
- Legend: `plugins.legend.display = true` when series.length > 1 OR kind is pie/donut (multi-slice); the
  labels come from Chart.js automatically (series[].name for bar/line/area datasets → set each dataset's
  `label = series.name`; pie/donut labels come from `data.labels`). Single-series bar/line/area may keep
  legend hidden (matches 4.1 behavior). Read all colors via getComputedStyle — NO raw color literals except
  the transparent/fill-alpha handling for area (derive from the resolved token, e.g. Chart.js fill uses the
  same stroke color; keep it token-derived).
- Redraw-in-place (.update()), canvas reuse, and the mark-sweep/destroy of dropped charts MUST be preserved
  exactly as today — only the config construction + controller registration change.
</rendering_spec>

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-05 | Tampering (injection) | chart labels/title/series names | mitigate | Chart.js renders text to CANVAS (not innerHTML), so label/title strings cannot inject markup; no `innerHTML` sink is added for chart text |
| T-18-06 | Denial of service | app with no chart paying chart.js cost | mitigate | The `import("chart.js")` stays DYNAMIC/lazy — an app without a ChartNode loads zero chart.js bytes (asserted: not converted to a static import) |
| T-18-07 | Information disclosure | raw color leaking to the wire | mitigate | All color is resolved from CSS tokens inside the adapter via getComputedStyle; no color literal crosses the wire |
| T-18-SC | Tampering | package installs | accept | No new installs — additional controllers come from the pre-existing chart.js `^4` optional dep |
