# Phase 5 — UI Design Contract

> Visual and interaction contract for the canonical-examples + 0.4.0 release-closeout phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase nature (read first):** This phase ships **no new design system, no new wire/model field, no new component, and no new CSS rules**. The ViewModel Shell design system already shipped in Phases 3–4: the `--vms-*` spacing/type scale tokens, `density`, `section variant:"card"`, the `.vms-page` centered shell with `--vms-page-max`, the `layout` preset enum (`stack`/`split`/`cards`), the 11 `:root` theme files, and the additive `--vms-*` override seam. This contract therefore **documents how to compose the canonical archetypes on the EXISTING shipped tokens/presets/themes**, plus the **visual-quality acceptance contract** (Bootstrap benchmark mapping + falsifiable structural CI proxies + a documented reviewer sign-off). Appearance is 100% the shipped CSS. The override seam is sacred: additive `--vms-*` only; the 11 theme files stay byte-identical (D-03). The only value changes in the entire phase are (a) the `default.css` `:root` dark→light re-baseline (D-01) and (b) the new byte-exact `themes/dark-purple.css` capture file (D-02).

## Spacing Scale

The shipped spacing scale (`default.css` `:root`, lines ~42–48). 6 steps, base `--vms-space-xs` 0.5rem, modular ratio ~1.5 (decided Phase 3 03-01, D-06/D-07). **This phase declares no new spacing tokens and authors no spacing literals.** The 4px-multiple template default does not apply — the shipped scale is rem-based and is the contract. Note `--vms-space-2xs` (0.25rem = 4px) is the smallest step; the rest are not 4px multiples by design and that is correct and locked.

| Token | Value | px @16 root | Usage (as shipped) |
|-------|-------|-------------|--------------------|
| `--vms-space-2xs` | 0.25rem | 4px | Field label↔input gap, tab gaps, tight inline padding |
| `--vms-space-xs` | 0.5rem | 8px | List gaps, stat-bar item gaps |
| `--vms-space-sm` | 0.75rem | 12px | Section/form gaps, control vertical padding |
| `--vms-space-md` | 1rem | 16px | Default element spacing, card padding, control horizontal padding |
| `--vms-space-lg` | 1.5rem | 24px | Page gap, split-column gap, stat-bar spacing |
| `--vms-space-xl` | 2.25rem | 36px | Select caret padding-right, largest rhythm step |

Density remap (`comfortable` default vs `compact` via `PageNode.density`, shipped Phase 3): `.vms-page--compact` remaps `--vms-space-sm`→0.5rem, `--vms-space-md`→0.75rem, `--vms-space-lg`→1rem. Archetype views render the **comfortable** default (no `density` set) unless an archetype explicitly teaches `compact` (Claude's discretion within D-10; if used, set it via the model `density` field, never via CSS).

**Exceptions:** The sanctioned single-token seam (D-17) permits a per-app `:root{}` override of **`--vms-page-max` and `--vms-font-*` only** — applied as a tiny per-app stylesheet imported in `main.ts` after the pinned theme (D-19), never as an HTML `<style>` and never any other token. No spatial/utility classes, no `m-*`/`p-*`, no column spans (REQUIREMENTS Out of Scope guard).

## Typography

The shipped type scale (`default.css` `:root`, lines ~50–56). 7 steps, all rem, base `--vms-text-base` 0.8125rem, modular ratio ~1.2 (decided Phase 3 03-01, D-06/D-09). Two weights only (regular 400 + semibold 600 — the only weights authored anywhere in `default.css`). The 3–4-size template guidance is superseded: the shipped 7-step scale is the locked contract; archetypes select from it, they do not add sizes or weights.

| Token | Value | px @16 root | Weight | Line height | Role (as shipped) |
|-------|-------|-------------|--------|-------------|-------------------|
| `--vms-text-xs` | 0.6875rem | 11px | 600 | inherit | Section headings (uppercase, tracked), table `th`, stat-bar labels |
| `--vms-text-sm` | 0.75rem | 12px | 400 | inherit | Field labels, secondary/muted text, tab labels |
| `--vms-text-base` | 0.8125rem | 13px | 400 | 1.4 (`.vms-text`), 1.5 (code) | Body default, buttons, table cells, error text |
| `--vms-text-md` | 0.875rem | 14px | 400 | inherit | Form inputs |
| `--vms-text-lg` | 1rem | 16px | 600 | 1.6 (`.vms-text--body`) | Subheadings, stat values |
| `--vms-text-xl` | 1.375rem | 22px | 600 | 1.2 | `.vms-text--heading`, modal titles |
| `--vms-text-2xl` | 2.25rem | 36px | 600 | 1.2 | Page title (`.vms-page__title`) |

Weights: regular **400** (body, labels, inputs, default buttons) and semibold **600** (page/section/modal titles, subheadings, stat values, primary CTA, table `th`). No other weights. Line heights as shipped: body prose 1.5–1.6, default `.vms-text` 1.4, headings/title ~1.2 (`letter-spacing: -0.02em` on page title). Archetypes inherit these — no per-archetype typography overrides.

## Color

Color is delivered entirely by the active shipped theme file's `:root` block. **The 0.4.0 shipped default re-bases dark→light onto the existing `light-purple.css` value set (D-01).** Values below are the new shipped default (verbatim from `light-purple.css` lines 6–23, which becomes `default.css` `:root` color block). `light-purple.css` itself stays byte-unchanged (becomes a harmless no-op override — D-02). The prior dark default is preserved byte-exact in the new `themes/dark-purple.css` (D-02).

### New shipped default (light-purple value set, D-01)

| Role | Token | Value | Usage (as shipped) |
|------|-------|-------|--------------------|
| Dominant (~60%) | `--vms-bg` | `#f7f7f9` | Page/body background |
| Dominant surface | `--vms-surface` | `#ffffff` | Cards, list items, inputs, modal, table header zone |
| Secondary (~30%) | `--vms-surface-2` | `#f0f0f4` | Default/secondary buttons, table `th`, checkbox mark, active tab |
| Secondary | `--vms-border` | `#d8d8e0` | All borders, dividers, table grid |
| Accent (~10%) | `--vms-accent` | `#5a4ad7` | See reserved-for list below |
| Accent support | `--vms-accent-glow` | `rgba(90,74,215,0.10)` | Focus ring, secondary-button hover bg |
| Accent support | `--vms-accent-dim` | `rgba(90,74,215,0.30)` | Hover border on buttons/list items |
| Text | `--vms-text` | `#1a1a22` | Body text on bg/surface |
| Text muted | `--vms-text-muted` | `#6c6c80` | Labels, captions, section headings, placeholders |
| Destructive | `--vms-error` | `#c2453d` | Destructive actions / error state only |
| Destructive support | `--vms-error-glow` | `rgba(194,69,61,0.10)` | Error banner bg, danger-button hover bg |
| Semantic | `--vms-warning` | `#c89610` | `list-item--warning`, table row warning tint |
| Semantic | `--vms-priority-high` | `#d76410` | `list-item--high` left accent |
| Semantic | `--vms-success` | `#2da359` | `list-item--success` left accent |
| Semantic | `--vms-info` | `#2277dd` | `list-item--info` left accent, links in tables |
| Done state | `--vms-done-bg` / `--vms-done-text` | `#ececf2` / `#9090a0` | Completed list items / strikethrough text |
| Color scheme | `--vms-color-scheme` | `light` | Native form-control chrome hint (date/select/scrollbar) |

**Accent (`--vms-accent`) reserved for:** primary-button background; focus ring/border on inputs (`--vms-accent-glow`); active-tab text + sort-indicator glyph; links (`.vms-link`, `.vms-table__link`); stat-bar values; progress bar fill; standalone checkbox checked state. **Never** general body text, never every border, never section/container backgrounds. (This is the shipped allocation — do not broaden it in archetypes.)

## Demo De-Chroming Contract (EXAMPLES-02, D-15/D-16/D-17/D-18/D-19)

Every demo HTML shrinks to a minimal scaffold with **zero `<style>`**: `<!doctype>` + `<head>` (meta/title; font `<link>` kept **only** where a `--vms-font-*` override is kept) + `<body><div id="app"></div><script>`. The shipped `styles.css` + the demo's one pinned theme are imported via `main.ts`/`server.ts` (the Showcase pattern). The `.vms-page` shell + `default.css` body rule own all reset/centering/width/background/font — making every demo's current inline `*{box-sizing}`, `body{flex centering}`, `#app{max-width}` redundant.

| Current per-demo chrome | 0.4.0 disposition |
|-------------------------|-------------------|
| `*{box-sizing/margin/padding}` reset | **Dropped** — `.vms-page` subtree box-sizing is shipped; body chrome is the `default.css` body rule |
| `body{background/color/font/flex centering/padding}` | **Dropped** — `default.css` body rule + `.vms-page` centered shell own this |
| `#app{max-width:520/660px}` | **Dropped**; if a narrower page is genuinely wanted, set **only `--vms-page-max`** via the sanctioned single-token seam (D-17) |
| `body.is-loading #app{opacity}` dispatch-dim affordance | **Dropped + logged as a deferred framework-gap idea** (no framework expression in 0.4.0 — D-15; not hand-patched) |
| Tasks `.vms-form{flex-direction:row}` horizontal add-form | **Dropped, renders default vertical**; gap logged as a deferred usage-driven model idea (no new wire in a closeout phase — D-16) |
| HelpDesk per-role inline `:root` theme blocks (`agent.html`/`requester.html`) | **Replaced** by two distinct shipped-theme imports via the seam (D-08/D-18) — role differentiation stays semantic, via the sanctioned seam |
| Branded font `<link>` | **Kept only if** a matching `--vms-font-*` token override is kept (D-17) |

Sanctioned single-token override (D-17/D-19): the **only** allowed per-app deviation is a tiny per-app stylesheet containing **one `:root{}` with only `--vms-page-max` and/or `--vms-font-*`**, imported in `main.ts` immediately **after** the pinned theme import (same mechanism the Showcase uses) — **never** an HTML `<style>`. This is the taught retune path ("override the token, don't hand-roll"). Exact filename/path is Claude's discretion (D-19); record it in the verification doc.
