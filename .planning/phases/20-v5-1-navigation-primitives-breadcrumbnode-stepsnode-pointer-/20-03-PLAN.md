private link(n: LinkNode, parent: HTMLElement): void {
  const a = document.createElement("a");
  a.className = n.active ? "vms-link vms-link--active" : "vms-link";
  a.href = n.href;
  a.textContent = n.label;              // ← textContent, never innerHTML (XSS-safe)
  if (n.active) a.setAttribute("aria-current", "page");
  if (n.external) { a.target = "_blank"; a.rel = "noopener noreferrer"; }
  parent.appendChild(a);
}

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| app-supplied crumb/step strings → DOM | An app-authored `label`/`description`/`href` string is written into the DOM by the renderer |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-05 | Tampering (XSS) | breadcrumb()/steps() text rendering | mitigate | Set all `label`/`description` via `textContent` and `href` via the `a.href` property (never `innerHTML`), exactly the link() precedent — an app string cannot inject markup |
| T-20-06 | Elevation of Privilege | external crumb link | mitigate | `external:true` crumbs set `rel="noopener noreferrer"` + `target="_blank"` (verbatim from link()) so a linked page cannot reach back via `window.opener` |
| T-20-07 | Information Disclosure | insufficient marker contrast | mitigate | Hand-check white-on-`--vms-accent` marker fill clears WCAG-AA in default + all 12 themes (the automated PAIRS list misses it) so step state is legible, not conveyed by color alone (also carried by per-marker `aria-label`) |

No package-manager installs in this plan → no legitimacy gate needed.
</threat_model>
