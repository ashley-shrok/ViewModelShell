<interfaces>
The capability seam (from viewmodel-shell/src/index.ts after Plan 01 — Adapter interface):

```
export interface Adapter {
  render(vm, onAction): void;                                              // required
  navigate?(url: string): void;                                            // optional verb
  storage?(scope: "local" | "session", key: string, value: string): void;  // optional verb
  transport?(input, init, hooks?): Promise<Response>;                       // optional override; fetch is default
}
```

Redirect resolution order (D-05): `ShellOptions.onRedirect` (if set, signature unchanged `(url: string) => void`) -> `adapter.navigate` -> loud error (D-06: never a silent no-op — a swallowed auth-token storage write or a no-op'd redirect is a correctness/security failure).

Enforcement: `viewmodel-shell/scripts/check-core-platform-globals.mjs`, runnable via `npm run check:core-globals`, gating step in `.github/workflows/parity.yml`; scope is `src/index.ts` only (browser.ts excluded — it legitimately owns DOM bindings).

AGENTS.md current claim to augment: "The core never references HTMLElement, document, or any platform type." This stays true and gains a "here is how it's enforced" explanation. README.md is public/consumer-facing (npm landing); its "Server-initiated redirect" section already shows `onRedirect` — that documented behavior must remain accurate (do NOT imply onRedirect changed).
