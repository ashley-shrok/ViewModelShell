<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| wire JSON → renderer | The browser adapter reads `n.layout` off a deserialized node and decides whether to interpolate a CSS class name into `element.className` |
| stylesheet → browser layout engine | `default.css` rules consume the emitted modifier class; CSS values are static authored strings, not data-derived |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering / Injection (class-name reflection) | `browser.ts` `page()`/`section()` className template literal | mitigate (by-design, closed equality guard) | The interpolation is gated by `n.layout === "split" || n.layout === "cards"` — a CLOSED two-literal equality check. Only the strings `vms-page--split` / `vms-page--cards` (and section equivalents) can ever reach `className`; any other `n.layout` value (including a malformed/attacker-supplied string that slipped past the type at runtime, or `"stack"`) yields the empty string. The open-interpolation anti-pattern (`n.layout ? ... ${n.layout}` ) is explicitly prohibited in the action and asserted absent in acceptance criteria. No `high`/`medium` residual. |
| T-04-02 | Tampering | `default.css` modifier rules | accept | All CSS values are static author-written constants (`16rem`, `calc(50% - var(--vms-space-lg))`, `repeat(auto-fit, ...)`). No value is derived from wire/user input — there is no data path into the stylesheet. No residual risk. |
| T-04-03 | Denial of Service (layout) | grid track sizing on adversarial child counts | accept | A page/section with many children under `split`/`cards` produces normal CSS Grid reflow (bounded by the browser's existing layout engine). No new amplification: the same children already render under flex-column today. Not a meaningful DoS surface. |

**Severity conclusion:** No `high` or `medium` threats. The single security-relevant
concern — reflecting an attacker-controlled string into a CSS class name — is
*mitigated by design* by the closed two-literal equality guard in the renderer
(asserted present, with the open-interpolation form asserted absent). No new parsing,
network, file, or exec surface is introduced.
</threat_model>
