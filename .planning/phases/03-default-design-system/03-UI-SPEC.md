## Spacing Scale

**Token model (D-06, D-07):** New `--vms-space-*` CSS variables in `:root`. | Token | rem | px @16 | Snaps which current literals | Usage |

## Page Shell (THEME-01 â€” D-11, D-12, D-13, D-14)

The centered container is the **existing `.vms-page` rule itself**. No DOM node, no `browser.ts` change, no new emitted class (zero AGENTS.md CSS-table churn for the shell). It coexists with the existing flex-column `gap` rhythm on that element.

| Property | Value | Rationale |
|----------|-------|-----------|
| `--vms-page-max` (new `:root` var) | `1080px` | Within "â‰ˆ1080px" (D-12); comfortable line length for forms/tables, exposed so themes/apps retune via the seam |
| `max-width` | `var(--vms-page-max)` | Centered max-width container |
| `margin-inline` | `auto` | Horizontal centering |
| `padding-inline` | `clamp(1rem, 5vw, 2.25rem)` | Responsive horizontal padding, **zero media queries** (D-13). | `background` | **transparent (unchanged)** | `.vms-page` stays transparent; `body` keeps carrying `--vms-bg`/color/font exactly as today (D-14). Color behavior byte-identical â†’ THEME-05-safe; avoids double-surface nesting against `.vms-section--card`. |

## Density Modifier (THEME-03 â€” D-01, D-04, D-10)

**Model surface:** `PageNode.density?: "comfortable" | "compact"` â€” closed union (D-03), additive optional field on `src/index.ts` (`PageNode`, ~line 59), `viewmodel-shell-dotnet/ViewModels.cs` (`PageNode` record, ~line 99). **CSS implementation (D-10):** `.vms-page--compact` **remaps a small set of scale variables** (not per-rule overrides â€” D-06's variable architecture is the prerequisite). |-------|----------------------|----------------------------|---|
